# Quest Design



게임 퀘스트 설계를 위한 실행 스킬이다.



## 목적



이 스킬은 다음 작업에 사용한다.



- 메인 퀘스트 라인 설계

- 사이드 퀘스트 설계

- 목표, 단계, 완료 조건 정의

- 보상 구조 설계

- 실패 조건 및 대안 경로 설계

- 퀘스트 흐름도와 보상 밸런스 문서화

## 작업 원칙



- 세계관 설정을 먼저 읽고 작업한다.

- 단순 심부름 퀘스트를 피한다.

- 모든 퀘스트에는 내러티브적 의미가 있어야 한다.

- 메인 퀘스트는 긴장-이완-반전의 리듬을 가진다.

- 사이드 퀘스트는 메인 스토리를 다른 각도에서 비춘다.

- 보상은 행동의 결과처럼 느껴져야 한다.



## 작업 절차



1. 전체 퀘스트 규모를 정한다.

2. 메인 퀘스트 체인을 설계한다.

3. 메인 퀘스트를 보완하는 사이드 퀘스트를 설계한다.

4. 각 퀘스트의 시작 조건, 단계, 완료 조건을 정의한다.

5. 분기 포인트와 실패 상태를 정의한다.

6. 보상 체계와 난이도 흐름을 정리한다.

7. 대사 작성과 분기 설계에 전달할 요소를 정리한다.

## 품질 기준



- 퀘스트의 목표와 동기가 명확해야 한다.

- 메인/사이드 퀘스트의 역할이 구분되어야 한다.

- 보상은 시간과 난이도 대비 설득력 있어야 한다.

- 퀘스트가 세계관과 인물 설정을 활용해야 한다.

- 이후 대사/분기 설계가 가능할 정도로 정보가 충분해야 한다.



## 에러 핸들링



### 세계관이 없는 경우

- 사용자 요청에서 장르와 톤을 추론한다.

- 최소한의 설정을 가정하고 그 가정을 문서에 남긴다.



### 퀘스트 수가 너무 많아지는 경우

- 메인 3~5개, 사이드 3~5개 범위로 우선 설계한다.

- 추가 분량은 확장안으로 분리한다.
