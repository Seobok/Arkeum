# Quest Design Patterns



퀘스트 설계에 사용하는 방법론 문서다.



## 왜 패턴이 필요한가



퀘스트는 서사와 게임플레이를 연결하는 핵심 구조다.  

단순한 반복 과제를 피하고, 다양한 구조적 패턴을 조합해 플레이어 동기를 유지해야 한다.



## 퀘스트 아키타입 12가지



### 전투/행동 계열



| # | 아키타입 | 핵심 루프 | 서사적 기능 | 위험 요소 |

|---|---------|---------|-----------|----------|

| 1 | **사냥** | 특정 대상 처치 | 위협 제거, 영웅성 증명 | 단순 반복이 되기 쉬움 → 타깃에 스토리 부여 |

| 2 | **방어** | 거점/NPC 보호 | 책임감, 시간 압박 | 실패 시 좌절감 → 부분 성공 옵션 제공 |

| 3 | **추격** | 도망치는 대상 추적 | 긴장감, 시공간 활용 | 반복 실패 → 단계별 단서로 난이도 조절 |



### 탐색/발견 계열



| # | 아키타입 | 핵심 루프 | 서사적 기능 | 위험 요소 |

|---|---------|---------|-----------|----------|

| 4 | **탐험** | 미지 영역 발견 | 세계관 확장, 호기심 | 방향감 상실 → 환경 스토리텔링으로 유도 |

| 5 | **수집** | 아이템/정보 모으기 | 세계 이해 심화 | 지루한 심부름 → 각 아이템에 미니 서사 |

| 6 | **수수께끼** | 퍼즐/암호 해결 | 지적 성취감, 세계 규칙 학습 | 막힘 → 힌트 시스템 단계적 제공 |



### 사회/관계 계열



| # | 아키타입 | 핵심 루프 | 서사적 기능 | 위험 요소 |

|---|---------|---------|-----------|----------|

| 7 | **호위** | NPC와 동행 | 관계 형성, 캐릭터 발전 | 느린 NPC → AI 행동 품질 확보 |

| 8 | **협상** | 대화로 해결 | 세력 관계 탐구, 롤플레이 | 정답이 뻔함 → 모든 선택에 대가 부여 |

| 9 | **배달** | 물건/정보 전달 | 세계 이동, NPC 네트워크 | 단순 이동 → 이동 중 이벤트 삽입 |



### 변형/복합 계열



| # | 아키타입 | 핵심 루프 | 서사적 기능 | 위험 요소 |

|---|---------|---------|-----------|----------|

| 10 | **잠입** | 발각 없이 목표 달성 | 긴장감, 대안적 플레이 | 발각=실패 금지 → 발각 시 전투 전환 |

| 11 | **경쟁** | NPC/플레이어와 경쟁 | 성취감, 사회적 비교 | 불공정 → 시각적 진행 표시 |

| 12 | **딜레마** | 양립 불가 선택 | 도덕적 무게, 플레이어 정체성 | 정답 존재 → 모든 선택이 일장일단 |



## 퀘스트 구조 설계: 3층 아키텍처



```text

[표면층] 플레이어가 보는 것: "마을 장로가 약초를 구해달라고 한다"

[기계층] 게임이 추적하는 것: 이동→수집→전투→귀환 + 플래그 변경

[서사층] 이야기가 전달하는 것: 장로의 숨겨진 과거, 약초의 진짜 용도

```



### 표면층 설계 규칙

- 목표가 즉시 이해되어야 한다.

- 목표 달성 경로가 2가지 이상 있으면 좋다.

- 진행 상황이 눈에 보여야 한다.



### 기계층 설계 규칙

- 퀘스트 스텝은 보통 3~7단계 범위로 유지한다.

- 각 스텝은 가능한 한 다른 게임 메카닉을 활용한다.

- 실패 상태와 실패 후 처리 방식을 정의한다.



### 서사층 설계 규칙

- 퀘스트 완료 후 플레이어가 세계에 대해 새로운 것을 최소 1개 알게 한다.

- NPC 요청에는 표면적 이유와 숨은 이유를 구분한다.

- 퀘스트 결과가 후속 퀘스트에 영향을 미치게 한다.



## 보상 심리학: DRIP 모델



| 요소 | 설명 | 적용 |

|------|------|------|

| **D**elayed | 지연 보상 | 메인 퀘스트 보상은 챕터 끝에 |

| **R**andom | 랜덤 보상 | 전리품 확률 시스템 |

| **I**ntrinsic | 내적 보상 | 스토리, 관계 변화, 감정적 보상 |

| **P**rogressive | 점진 보상 | 칭호, 성장 트랙, 누적 성취 |



## 보상 밸런스 공식



```text

퀘스트 보상 가치 = (소요 시간 x 기본 보상률) x 난이도 계수 x 선택 보너스

```



## 난이도 곡선 설계



### 이상적 난이도 곡선: 톱니 패턴



```text

난이도

&#x20; ↑

&#x20; │    /      /        /

&#x20; │   /      /        /  

&#x20; │  /      /        /    

&#x20; │ /      /        /      

&#x20; │/                /        

&#x20; └───────────────────────────→ 진행도

```



- 각 꼭짓점은 보스전 또는 핵심 분기

- 각 골짜기는 회복, 탐색, 서사 전개 구간

- 전체 기울기는 점진적으로 상승



## 퀘스트 템플릿



```text

### 퀘스트: [제목]

- ID: Q_[챕터]_[번호]

- 유형: 메인 / 사이드 / 히든

- 아키타입: [12가지 중 선택]

- 선행 퀘스트: Q_XX_XX

- 추천 레벨: Lv.XX



#### 개요

[1~2문장 요약]



#### 트리거 조건

[퀘스트 시작 조건]



#### 스텝

1. [행동] — [게임 메카닉] — [서사 기능]



#### 분기

- 선택 A: [결과] → [후속 영향]

- 선택 B: [결과] → [후속 영향]



#### 보상

- 경험치: X / 골드: X / 아이템: [이름]

- 서사 보상: [관계 변화, 새로운 정보]



#### 플래그 변경

- SET: [flag_name] = [value]

```
