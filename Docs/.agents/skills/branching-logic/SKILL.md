# Branching Logic



게임 분기 설계를 위한 실행 스킬이다.



## 목적



이 스킬은 다음 작업에 사용한다.



- 스토리 핵심 분기점 설계

- 엔딩 구조 설계

- 플래그 시스템 설계

- 선택 결과의 즉시/장기 영향 설계

- 분기 후 합류 구조 설계

- 리플레이 가치 설계

## 작업 원칙



- 세계관, 퀘스트, 대사를 함께 참조한다.

- 환상적 선택을 최소화한다.

- 주요 선택에는 트레이드오프가 있어야 한다.

- 분기 수는 관리 가능한 범위로 유지한다.

- 플래그는 누적형 구조를 우선 고려한다.

- 엔딩은 과정의 반영이어야 한다.



## 작업 절차



1. 분기 설계 범위를 정의한다.

2. 핵심 분기점 후보를 추린다.

3. 각 분기점의 상황, 선택지, 즉시 결과, 장기 결과를 정리한다.

4. 플래그와 조건식을 설계한다.

5. 분기 후 합류 구조를 설계한다.

6. 엔딩 구조를 정의한다.

7. 리플레이 가치와 회차 차이를 정리한다.

8. 조건 충돌과 과복잡도를 점검한다.

## 품질 기준



- 각 핵심 선택이 실제로 결과를 바꿔야 한다.

- 엔딩은 누적 선택의 반영이어야 한다.

- 특정 루트만 압도적으로 유리하면 안 된다.

- 분기 수가 과도하게 폭증하면 안 된다.

- 합류 구조가 있어도 선택의 흔적이 남아야 한다.

- 플래그 시스템은 추적 가능하고 모순이 없어야 한다.

- 리플레이 동기가 존재해야 한다.



## 에러 핸들링



### 분기가 너무 복잡해지는 경우

- 핵심 분기점은 3~5개 수준으로 줄인다.

- 나머지는 flavor choice나 국소 반응으로 축소한다.



### 엔딩 조건이 모순되는 경우

- 플래그 충돌을 명시한다.

- 우선순위 규칙을 정의한다.

- 상호 배타 조건과 누적 조건을 분리한다.



### 선택의 차이가 너무 약한 경우

- 즉시 결과 또는 장기 결과를 강화한다.

- 관계, 평판, 접근 경로, 보상, 엔딩 조건 중 최소 하나에 실질 차이를 만든다.
