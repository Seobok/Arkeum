# Dialogue Systems



게임 대사 작성에 사용하는 방법론 문서다.



## 왜 게임 대사는 다른가



게임의 대사는 플레이어가 선택하고, 그 선택이 관계와 진행에 영향을 줄 수 있다.  

따라서 문장 자체뿐 아니라 선택지 구조, 감정 태그, 반응 흐름이 중요하다.



## 캐릭터 보이스 설계: VOICE 프레임워크



| 요소 | 설명 | 설계 질문 |

|------|------|----------|

| **V**ocabulary | 어휘 수준과 범위 | 어떤 단어를 자주 쓰고 어떤 단어를 피하는가 |

| **O**pinion | 세상에 대한 태도 | 낙관적/냉소적/실용적/이상주의적 중 어디에 가까운가 |

| **I**diom | 말버릇, 구두점 패턴 | 특유의 끝맺음, 습관적 표현, 반복어가 있는가 |

| **C**adence | 문장 리듬과 길이 | 짧고 빠른가, 길고 우아한가 |

| **E**motion | 감정 표현 방식 | 감정을 직접 말하는가, 억누르는가, 행동으로 드러내는가 |



### 캐릭터 보이스 카드 예시



```text

캐릭터: 노련한 용병대장

- V: 군사 용어 다수, 감정 어휘 최소

- O: 현실주의, 이상주의 조롱

- I: 문장 끝에 "...그래" 습관

- C: 짧은 문장 중심

- E: 감정을 거의 직접 표현하지 않음

```



## 선택지 설계 심리학



### 선택지 유형



| 유형 | 설명 | 서사 기능 | 예시 |

|------|------|----------|------|

| **성격 표현** | 플레이어의 롤플레이 | 캐릭터 정체성 구축 | 친절/냉정/유머러스 응답 |

| **정보 수집** | 추가 정보 획득 | 세계관 탐구 | "그게 무슨 뜻이지?" |

| **도덕적 딜레마** | 양립 불가 가치 충돌 | 감정적 무게 | 한 명만 구할 수 있다면? |

| **전략적 선택** | 게임플레이 영향 | 리소스/경로 결정 | 정면돌파/잠입/협상 |

| **관계 선택** | NPC 관계 변화 | 사회적 시뮬레이션 | 동의/반대/회피 |



### 선택지 설계 규칙



1. 선택지를 읽으면 대략적인 결과가 예측 가능해야 한다.

2. 모든 선택지가 단순히 착함/나쁨으로만 갈리지 않게 한다.

3. 보통 2~4개, 이상적으로는 3개 선택지를 사용한다.

4. 결과가 완전히 같은 거짓 선택을 피한다.

5. 침묵이나 시간 초과도 의미 있는 선택이 될 수 있다.



### 선택의 무게감



```text

무게감 = 되돌릴 수 없는 정도 x 영향 범위 x 감정적 투자

```



## 바크 시스템



바크는 비대화 상황에서 자동으로 발화되는 짧은 대사다.



### 바크 카테고리



| 카테고리 | 트리거 | 예시 |

|---------|--------|------|

| **유휴** | 플레이어 근처 대기 | "오늘 시장은 한산하군..." |

| **반응** | 플레이어 행동 | "이런, 조심해!" |

| **환경** | 날씨/시간/장소 변화 | "비가 오려나..." |

| **관계** | 호감도 변화 | "영웅이시여!" / "흥, 또 왔나" |

| **스토리** | 퀘스트 진행 | "장로님이 찾으시더라" |

| **전투** | 전투 상황 | "뒤에서 온다!" |



### 바크 품질 규칙

- 같은 바크 반복을 피하기 위해 변형을 여러 개 준비한다.

- 캐릭터 보이스를 유지한다.

- 필요하면 세계관 정보나 퀘스트 힌트를 자연스럽게 포함한다.

- 길이는 1~2문장 정도로 유지한다.



## 컷신 대사 연출



### 컷신 유형별 대사 밀도



| 컷신 유형 | 대사 비율 | 무언 연출 비율 | 목적 |

|----------|---------|-------------|------|

| 오프닝 | 30% | 70% | 세계관 몰입 |

| 캐릭터 소개 | 60% | 40% | 인물 각인 |

| 전투 전 | 40% | 60% | 긴장감 |

| 반전/트위스트 | 50% | 50% | 충격과 여운 |

| 엔딩 | 40% | 60% | 감정 정리 |



### 대사-침묵 리듬



```text

[대사] — [리액션 숏 1~2초] — [대사] — [긴 침묵 3~5초] — [결정적 대사]

```



## 대사 포맷 표준



```text

DIALOGUE_ID: D_[챕터]_[씬]_[순번]

SPEAKER: [캐릭터명]

EMOTION: [neutral/happy/angry/sad/fearful/surprised/contempt]

CONDITION: [IF flag_xxx == true]

VOICE_NOTE: [연기 지시]



[대사 내용]



→ CHOICE

&#x20; A: [선택지 텍스트] → GOTO ... / SET flag_xxx

&#x20; B: [선택지 텍스트] → GOTO ... / SET flag_xxx

```
