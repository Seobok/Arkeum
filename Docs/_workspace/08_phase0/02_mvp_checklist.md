# MVP 체크리스트

## 문서 목적
- `07_development_roadmap.md`의 0단계 산출물 중 `MVP 체크리스트`를 독립 문서로 정리한다.
- `첫 플레이 가능한 빌드`에 무엇이 포함되고 무엇이 제외되는지 명확히 고정한다.

## MVP 정의
- 목표: `10분 이상 반복 플레이 가능한 코어 루프`를 검증하는 첫 빌드
- 플레이 흐름: `거점 출발 -> 던전 탐사 -> 전투/자원 획득 -> 사망 또는 종료 -> 귀환 제단 복귀 -> 잔광 정산 -> 재도전`
- 기준 캐릭터: `재를 뒤집어쓴 기사` 1명
- 기준 지역: `귀환 제단`, `잿빛 회랑`

## MVP 확정 범위

### 플레이어 행동 세트
- [ ] 이동
- [ ] 기본 공격
- [ ] 상호작용
- [ ] 아이템 사용
- [ ] 대기

### 전투 및 시간 진행
- [ ] 플레이어 1행동 = 시간 1단위 규칙 적용
- [ ] 적이 플레이어 행동 후 반응
- [ ] 피격 처리
- [ ] 사망 처리
- [ ] 적 처치 처리

### 코어 루프
- [ ] 거점에서 런 시작
- [ ] 던전 진입
- [ ] 탐사 중 자원 획득
- [ ] 사망 또는 종료 시 런 종료
- [ ] `귀환 제단` 복귀
- [ ] 정산 결과 반영 후 즉시 재도전 가능

### 자원 체계
- [ ] `혈편` 획득
- [ ] `혈편` 던전 내부 사용처 1개 이상
- [ ] `혈편` 런 종료 시 전량 소멸
- [ ] `잔광` 런 종료 정산
- [ ] `잔광` 거점 사용처 1개 이상

### 콘텐츠 최소량
- [ ] 거점 1개
- [ ] 던전 층 1개 또는 테스트 구역 2개
- [ ] 적 2종 이상
- [ ] 무기 타입 1종 이상
- [ ] 소비 아이템 1종 이상

### 거점 기능
- [ ] 출발 준비
- [ ] `잔광` 사용 기능 1개 이상
- [ ] 거점 NPC 대화 1명 이상
- [ ] 런 결과 확인

### 저장/진척
- [ ] 총 회귀 횟수 저장
- [ ] 최고 도달 층 저장
- [ ] 영구 재화 `잔광` 저장
- [ ] 최소 퀘스트/플래그 저장

### 서사/UX 전달
- [ ] 죽음이 완전 실패가 아니라 회귀임을 전달
- [ ] `혈편`과 `잔광`의 차이를 UI 또는 문구로 전달
- [ ] 장비 소실과 영구 진척 보존 차이를 전달
- [ ] 플레이어의 초반 목표를 한 줄 수준으로 제시

## MVP 제외 범위
- [ ] 추가 플레이어블 캐릭터 해금
- [ ] 세력 퀘스트 본편
- [ ] 기억 분기 본격 반영
- [ ] 중층 이후 지역
- [ ] 엔딩 판정
- [ ] 고급 컷신/연출
- [ ] 복잡한 성장 트리
- [ ] 세력별 상세 보상 밸런싱

## 문서 기준 확인 체크

### 용어 통일
- [ ] `혈편`은 런 내부 전용 재화로만 사용한다.
- [ ] `잔광`은 거점 영구 재화로만 사용한다.
- [ ] 회귀 지점은 `귀환 제단`으로 통일한다.
- [ ] 첫 지역 명칭은 `잿빛 회랑`으로 통일한다.
- [ ] 진행 규칙은 `행동 단위 기반 진행`으로 통일한다.

### 첫 빌드 기능 명시
- [ ] 첫 빌드 포함 기능이 문서에 모두 기록되어 있다.
- [ ] 제외 기능이 명시되어 있다.
- [ ] 팀이 동일한 범위를 사용한다.

## 확정안

### 플레이어 개인 동기
- `성혈 아래에 묻힌 자신의 진짜 이름과 마지막 죽음의 원인을 되찾기 위해 계속 하강한다.`

### 플레이어 초반 목표 문구
- UI/퀘스트 표기 최종안: `잿빛 회랑으로 내려가 네 이름에 반응한 잔광을 회수하라.`

### 1차 거점 기능 목록
- 출발 준비
- `잔광` 사용을 통한 `응고 지혈포` 시작 해금 1종
- 거점 장례자 대화
- 런 결과 확인

### 1차 `혈편` 사용처
- 던전 내부 상인 1명
- `응급 회복 소모품` 구매 1종만 구현

### 1차 `잔광` 사용처
- 시작 소모품 `응고 지혈포` 해금 1종만 구현
- 해금 후 모든 새 런은 기본 보급 1개를 들고 시작

## MVP 완료 판정
- [ ] 신규 플레이어가 별도 설명 없이 `거점 -> 던전 -> 사망 -> 회귀 -> 강화 -> 재도전` 흐름을 이해할 수 있다.
- [ ] 10분 이상 반복 플레이가 가능하다.
- [ ] 적어도 한 번의 재도전에서 누적 진척의 차이를 체감할 수 있다.
- [ ] 다음 구현 단계로 넘어가기 위한 피드백 포인트를 수집할 수 있다.
