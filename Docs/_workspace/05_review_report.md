# 리뷰 보고서

## 검토 범위
- 검토 대상:
  - `01_worldbuilding.md`
  - `02_quest_design.md`
  - `03_dialogue_script.md`
  - `04_branch_map.md`
- 검토 관점:
  - 설정 정합성
  - 퀘스트 동기와 반복 구조 적합성
  - 대사 톤 일관성
  - 분기 추적 가능성

## 총평
- 현재 문서 세트는 `회귀의 저주`, `하강을 통한 진실 접근`, `영구 재화의 서사적 정당화`, `캐릭터 해금의 해방 구조`를 비교적 일관되게 공유하고 있다.
- 메인 퀘스트, 대사, 분기 모두가 `낙죄의 성혈`을 단순 던전이 아니라 죄와 기억을 재구성하는 장치로 다루고 있어 세계관 중심축은 안정적이다.
- 세력 갈등도 단순 선악 구도가 아니라 회귀와 성혈의 해석 차이로 정리되어 있어 후속 확장 여지가 충분하다.

## 확인된 강점
- 아이템 소실, 회귀, 영구 재화가 하나의 설정 규칙으로 자연스럽게 묶여 있다.
- 캐릭터 해금이 구조적 보상에 머무르지 않고 각 인물의 속박 해소와 연결되어 있다.
- 엔딩 분기가 세력 우호도보다 세계 해석의 방향성에 기반해 있어 테마 일관성이 높다.
- 대사 톤이 세력과 인물 성격에 맞게 구분되어 있어 후속 장면 확장 시 기준점이 된다.

## 리스크 및 보완 필요 사항

### 1. 메인 퀘스트의 플레이 동기 선명도
- 문제:
  - `MQ-02` 이후 플레이어가 왜 계속 하강해야 하는지 정서적 동력이 다소 약해질 수 있다.
- 영향:
  - 세계 존속 문제는 크지만, 개인적 동기가 약하면 반복 탐사 구조에서 몰입이 흔들릴 수 있다.
- 제안:
  - `MQ-01` 또는 `MQ-04`에서 플레이어 자신의 상실 대상 하나를 더 선명하게 고정한다.
  - 예: 잃어버린 이름, 특정 인물, 이전 루프의 결말 기록.

### 2. 세력 퀘스트 보상 구분의 추가 필요
- 문제:
  - 현재 세력별 보상 방향은 분리되어 있으나, 실제 플레이 체감 차별화 기준은 더 필요하다.
- 영향:
  - 선택의 의미가 서사적으로는 살아도 체감상 비슷해질 수 있다.
- 제안:
  - 수도회는 침식 완화와 정보 공개
  - 흑철 잔존단은 생존성과 전투 교리
  - 연금회는 개조와 위험한 효율
  - 무명 계약자는 탐색/루프 변형
  - 위와 같이 플레이 성향 차이를 명시적으로 더 벌리는 것이 좋다.
- 보강 결과:
  - `02_quest_design.md`와 `06_system_specification.md`에 세력별 보상 차이 초안을 추가했다.
  - 현재 기준 초안은 `안정/힌트`, `전투/돌파`, `효율/개조`, `탐색/루프 변형`의 4축으로 정리된다.

### 3. 기억 관련 분기 설명의 단계적 공개 필요
- 문제:
  - `기억 보존`과 `기억 소거` 축이 세계관상 중요하지만, 초반부터 과도하게 제시되면 추상적으로 느껴질 수 있다.
- 영향:
  - 플레이어가 선택 의미를 이해하기 전에 철학적 선택만 남을 위험이 있다.
- 제안:
  - 초반은 개인 정체성 중심
  - 중반은 성혈 시스템 이해 중심
  - 후반은 윤리적 선택 중심
  - 이렇게 공개 단계를 나누는 편이 적합하다.

### 4. 엔딩 E의 활용도 관리 필요
- 문제:
  - `공허한 반복` 엔딩은 세계관적으로 타당하지만, 준비가 부족하면 단순한 미달 엔딩처럼 보일 수 있다.
- 영향:
  - 플레이어가 실패했다고만 느끼면 서사적 여운이 약해진다.
- 제안:
  - END-E에도 고유 컷신, 반복된 제단 대사 변화, 정체성 붕괴 연출을 넣어 독립된 결말처럼 보강한다.

## 문서 간 충돌 여부
- 명시적 충돌은 현재 기준으로 없음.
- 재화 체계는 이번 보강으로 `혈편`(런 내부 전용)과 `잔광`(거점 영구 전용)으로 정리되어, 기존의 다중 영구 재화 후보 상태보다 추적성이 높아졌다.
- 플레이어 개인 동기, `잔광` 1차 사용처, `혈편` 1차 사용처, 거점 MVP 기능, `MQ-01` 초반 목표 문구가 이번 보강으로 확정되어 초기 빌드 범위가 더 명확해졌다.
- 다음 항목은 후속 제작 시 주의가 필요하다.
  - `03_dialogue_script.md`에서 수도회 대사가 지나치게 진실을 직접 설명하면 `04_branch_map.md`의 후반 진실 공개 흐름이 약해질 수 있다.
  - `02_quest_design.md`의 캐릭터 해금 퀘스트가 너무 빠르게 완료되면 반복 탐사의 축적형 구조가 약해질 수 있다.
  - `04_branch_map.md`의 엔딩 판정은 세력 상태값보다 신념 플래그 우세를 중심으로 유지해야 현재 테마가 흔들리지 않는다.

## 추가 검토: 던전 맵/몬스터 턴 리팩터링 요구사항

### 검토 대상
- `10_dungeon_map_refactor_requirements.md`
- `11_monster_turn_refactor_requirements.md`

### 종합 판단
- 두 문서는 주로 구현 요구사항이므로 기존 세계관과 직접 충돌하는 설정은 많지 않다.
- `10_dungeon_map_refactor_requirements.md`의 다중 방/통로/문 구조는 `낙죄의 성혈`을 살아 있는 의식 시설이자 층별 기억 장치로 해석하는 현재 세계관과 잘 맞는다.
- `11_monster_turn_refactor_requirements.md`의 플레이어 행동 후 몬스터 턴 진행, 준비 후 공격/이동, 타겟 타일 예고는 `행동이 곧 시간`이라는 기존 규칙을 강화한다.

### 어색하거나 보강이 필요한 설정

1. `10_dungeon_map_refactor_requirements.md`의 랜덤 방 배치
   - 현재: 시작 방을 제외한 방은 랜덤하게 배치된다.
   - 리스크: 던전이 단순 절차 생성 공간처럼 보이면, `낙죄의 성혈`이 죄와 기억을 재구성하는 장치라는 설정이 약해질 수 있다.
   - 제안: 랜덤 배치는 "성혈이 회귀자의 기억과 죄업을 재배열한다"는 식으로 해석한다. 같은 층이라도 회귀마다 방 배열이 달라지는 이유를 `응답하는 심연`의 공간 반응으로 설명하면 자연스럽다.

2. `10_dungeon_map_refactor_requirements.md`의 최소 6개 방 규칙
   - 현재: 하나의 층에는 최소 6개의 방이 존재해야 한다.
   - 리스크: 숫자 자체가 서사적으로 드러나면 인위적으로 느껴질 수 있다.
   - 제안: 문서 내 구현값으로만 유지하고, 인게임에서는 "성혈의 층은 여러 기억실과 회랑으로 갈라진다" 정도로 표현한다. 특정 층의 의식 규칙으로 6이라는 숫자를 강조할 계획이 없다면 세계관 고정 설정으로 올리지 않는 편이 좋다.

3. `10_dungeon_map_refactor_requirements.md`의 문/통로 규칙
   - 현재: 방과 통로는 문을 통해서만 접하고, 같은 방향 문끼리는 연결하지 않는다.
   - 리스크: 물리 규칙으로만 보이면 타일 생성 제약처럼 느껴질 수 있다.
   - 제안: `봉인문`, `의식문`, `경계 지점`이 이미 주요 상징으로 쓰이고 있으므로, 일반 방 문도 "성혈 내부의 경계 장치"라는 톤을 유지한다. 단, 모든 문을 메인 퀘스트의 `봉인문`과 동일한 위상으로 보이게 만들면 중요도가 흐려질 수 있으므로 명칭을 구분한다.

4. `10_dungeon_map_refactor_requirements.md`의 통로 depth
   - 현재: 통로 depth는 층 번호 기반이다.
   - 리스크: 기존 세계관은 하강 자체가 진실 접근을 뜻하므로, 같은 층 내부 통로가 모두 같은 depth를 갖는 것은 큰 문제는 없다. 다만 후반부에서 현실과 기억의 경계가 무너지는 하층까지 동일 규칙을 적용하면 공간의 기괴함이 약해질 수 있다.
   - 제안: 초반/중반은 층 번호 기반으로 유지하고, `유리 성소` 이후에는 방 depth 또는 이벤트 depth를 섞을 수 있는 여지를 남긴다. 요구사항의 "추후 변경 가능" 메모는 서사 확장 측면에서도 적절하다.

5. `11_monster_turn_refactor_requirements.md`의 차폐 무시 탐지
   - 현재: 탐지 범위는 구조물, 벽, 문 등의 차폐 여부와 무관하게 맨해튼 거리로 계산한다.
   - 리스크: 일반 몬스터가 벽 너머 플레이어를 항상 감지하면 물리적 공간감이 어색할 수 있다.
   - 제안: 모든 몬스터의 보편 감각으로 설명하기보다, 성혈 내부 존재들이 "소리나 시야"가 아니라 "행동의 박자", "피의 흔들림", "회귀자의 잔광"을 감지한다고 정리하면 기존 `응답하는 심연` 규칙과 잘 맞는다. 은신/차폐 플레이를 나중에 넣을 계획이 있다면 몬스터 타입별 예외를 열어둔다.

6. `11_monster_turn_refactor_requirements.md`의 준비 타일 시각화 색상
   - 현재: 공격 준비 타일은 초록색, 이동 준비 타일은 붉은색 계열 마커로 표시한다.
   - 리스크: 기존 세계관의 핵심 시각 언어가 성혈, 재, 유리, 성광이므로 초록색 공격 예고는 독성/부식 몬스터처럼 읽힐 수 있다.
   - 제안: 공격 예고는 성혈색, 재색, 성광색 등 세계관 팔레트와 맞추는 편이 낫다. 초록색을 유지한다면 "부패한 성광" 또는 특정 적 유형의 예고색으로 한정하는 것이 자연스럽다.

7. `11_monster_turn_refactor_requirements.md`의 빗나감 처리
   - 현재: 저장된 공격 타일에 플레이어가 없으면 공격은 빗나간 것으로 처리한다.
   - 리스크: 설정 충돌은 아니지만, 묵직한 다크 판타지 톤에서는 단순 miss보다 "성혈이 헛된 자리로 폭발한다", "기억 자국을 찢는다" 같은 연출이 더 어울린다.
   - 제안: 시스템 규칙은 그대로 두되, 연출 문구는 몬스터 종류와 층 테마에 맞춰 다르게 둔다.

### 문서 간 충돌 여부
- 명시적 충돌 없음.
- `10_dungeon_map_refactor_requirements.md`는 `06_system_specification.md`의 던전 진행 및 층 구조 시스템과 호환된다.
- `11_monster_turn_refactor_requirements.md`는 `01_worldbuilding.md`의 `행동에 반응하는 심연`, `06_system_specification.md`의 행동 기반 시간 진행 시스템과 호환된다.
- 주의할 점은 두 문서의 기술 용어가 그대로 인게임 텍스트로 올라오면 서사 톤이 건조해질 수 있다는 점이다. 구현 문서는 현재 형태를 유지하되, 대사/튜토리얼/연출 문서에서는 세계관 용어로 번역하는 기준이 필요하다.
