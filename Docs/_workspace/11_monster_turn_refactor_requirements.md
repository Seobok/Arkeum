# Monster Turn Refactor Requirements

## Purpose
- 플레이어 행동 이후 몬스터 턴이 진행되는 규칙을 명확히 정의한다.
- 몬스터의 탐지, 타겟 유지, 준비 상태, 이동력, 공격/이동 의사결정을 구현 기준으로 정리한다.
- 기존 단순 행동 주기 기반 AI를 확장 가능한 턴 기반 몬스터 행동 시스템으로 리팩터링한다.

## Player Action And Monster Turn
- 플레이어가 턴을 소비하는 행동을 하면 몬스터 턴이 진행된다.
- 턴을 소비하는 플레이어 행동은 다음을 포함한다.
  - 이동
  - 공격
  - 아이템 사용
  - 대기
  - 턴 소비로 판정되는 상호작용
- 플레이어 행동이 실패하거나 턴을 소비하지 않는 상호작용이면 몬스터 턴은 진행되지 않는다.

## Monster Detection And Targeting
- 몬스터는 공격 범위와 별개로 탐지 범위를 가진다.
- 탐지 범위는 구조물, 벽, 문 등의 차폐 여부와 무관하게 몬스터와 플레이어 사이의 맨해튼 거리로 계산한다.
- 맨해튼 거리 공식은 `abs(enemy.x - player.x) + abs(enemy.y - player.y)`이다.
- 플레이어가 탐지 범위 안에 들어오면 해당 플레이어가 몬스터의 타겟이 된다.
- 플레이어가 탐지 범위 밖으로 나가면 몬스터의 타겟은 초기화된다.
- 현재 구현 범위에서는 플레이어가 1명이라고 가정하되, 타겟 상태는 액터 식별자 기반으로 보관해 이후 다중 타겟 확장을 막지 않는다.

## Monster Preparation States
- 몬스터 행동은 즉시 실행 또는 준비 후 실행될 수 있다.
- 몬스터는 공격 준비 상태와 이동 준비 상태를 가진다.
- 공격 준비 상태의 몬스터는 공격 준비 지속 턴 수가 지난 뒤 공격 행동을 실행한다.
- 이동 준비 상태의 몬스터는 이동 준비 지속 턴 수가 지난 뒤 이동 행동을 실행한다.
- 준비 턴 수가 `0`이면 해당 행동은 현재 몬스터 턴에 즉시 실행된다.
- 준비 중이던 행동과 현재 선택해야 할 행동이 달라지면 기존 준비 상태는 취소되고 새 행동 준비를 시작한다.
  - 예: 이동 준비 중 플레이어가 공격 범위에 들어오면 이동 준비를 취소하고 공격 준비로 전환한다.
  - 예: 공격 준비 중 플레이어가 탐지 범위 밖으로 나가면 공격 준비를 취소하고 비타겟 이동 준비로 전환한다.

## Monster Movement
- 몬스터는 이동력을 가진다.
- 타겟이 존재하고 타겟이 공격 범위 밖에 있으면 몬스터는 플레이어를 향해 이동한다.
- 플레이어를 향해 이동할 때는 한 번의 이동 행동으로 최대 이동력만큼 이동한다.
- 이동 도중 다음 칸이 이동 불가 타일, 플레이어 위치, 또는 다른 생존 몬스터가 점유한 칸이면 그 앞에서 이동을 멈춘다.
- 타겟이 없을 때 몬스터는 상하좌우 중 이동 가능한 랜덤한 위치로 이동한다.
- 타겟이 없는 랜덤 이동은 이동력과 무관하게 1칸만 이동한다.
- 랜덤 이동에도 이동 준비 지속 턴 수 규칙은 적용된다.

## Monster Decision Order
1. 몬스터는 먼저 타겟인 플레이어가 존재하는지 확인한다.
2. 타겟이 없다면 이동 준비 규칙을 적용한 뒤 상하좌우 중 랜덤한 이동 가능 위치로 1칸 이동한다.
3. 타겟이 있다면 플레이어가 공격 범위 내에 있는지 확인한다.
4. 플레이어가 공격 범위 밖이면 이동 준비 규칙을 적용한 뒤 플레이어를 향해 최대 이동력만큼 이동한다.
5. 플레이어가 공격 범위 안이면 공격 준비 규칙을 적용한 뒤 플레이어를 공격한다.

## Data Model
- `ActorStats.DetectionRange`: 몬스터 탐지 범위.
- `ActorStats.MovementRange`: 타겟 추적 이동 시 한 번에 이동 가능한 최대 칸 수.
- `ActorStats.AttackPreparationTurns`: 공격 행동 전 필요한 준비 턴 수.
- `ActorStats.MovePreparationTurns`: 이동 행동 전 필요한 준비 턴 수.
- `ActorEntity.TargetActorId`: 현재 타겟 액터 식별자.
- `ActorEntity.PendingEnemyAction`: 현재 준비 중인 몬스터 행동.
- `ActorEntity.PendingEnemyActionTurns`: 현재 행동을 준비한 몬스터 턴 수.

## Behavior Tree Structure
- 몬스터 의사결정은 Behavior Tree로 구성한다.
- `EnemyTurnSystem`은 몬스터 턴의 반복 실행만 담당하고, 개별 몬스터의 판단은 Behavior Tree 루트 노드를 Tick하는 방식으로 처리한다.
- 기본 트리는 다음 순서로 동작한다.

```text
Sequence
  UpdateTarget
  Selector
    Sequence
      HasNoTarget
      WanderMove
    Sequence
      HasTarget
      FaceTarget
      Selector
        Sequence
          CanAttackTarget
          AttackTarget
        ChaseMove
```

- `Sequence`는 자식 노드가 모두 성공해야 성공한다. 중간에 실패하거나 실행 중이면 즉시 해당 결과를 반환한다.
- `Selector`는 성공 또는 실행 중인 첫 자식 결과를 반환한다. 모든 자식이 실패하면 실패한다.
- `Running` 상태는 준비 턴을 소모 중인 행동을 의미한다. 이 경우 같은 몬스터 턴 안에서는 다음 후보 행동으로 넘어가지 않는다.
- 조건 노드는 상태를 변경하지 않고 판단만 수행한다.
- 액션 노드는 타겟 갱신, 방향 전환, 준비 턴 누적, 공격, 이동처럼 실제 상태 변경을 수행한다.

## Behavior Tree Extension Rules
- 새 몬스터 행동은 새 액션 메서드 또는 새 노드로 추가한다.
- 몬스터별 AI 차이는 `EnemyBehaviorTreeFactory`에서 트리를 선택하거나 생성하는 방식으로 확장한다.
- 공격 패턴, 도망, 소환, 순찰, 상태 이상 반응 같은 규칙은 기존 `EnemyTurnSystem`에 if/else를 추가하지 않고 Behavior Tree 노드로 추가한다.
- 트리 실행에 필요한 런타임 의존성은 `EnemyBehaviorContext`를 통해 전달한다.
- 전투, 타겟팅, 맵 이동 판정 같은 실제 게임 규칙은 `EnemyBehaviorActions`에 모아 노드 구조와 행동 구현을 분리한다.
- 준비 턴 상태는 액션 구현에서 유지하며, 선택된 행동이 바뀌면 기존 준비 상태를 취소하고 새 행동 준비를 시작한다.

## Prepared Action Target Lock
- 공격 또는 이동 준비 상태에 들어갈 때는 준비 시작 시점에 정한 실행 위치를 저장해야 한다.
- 공격 준비 상태는 공격하려고 정한 타일을 저장한다.
- 이동 준비 상태는 이동하려고 정한 타일을 저장한다.
- 공격 또는 이동을 실제 실행할 때는 현재 플레이어 위치나 현재 최적 이동 경로를 다시 계산하지 않고, 저장된 타일을 기준으로 실행한다.
- 준비 중 행동 종류가 바뀌면 저장된 타일도 초기화하고 새 행동의 타일을 다시 저장한다.
- 준비 중 타겟이 탐지 범위 밖으로 나가거나 행동 조건이 무효가 되면 저장된 타일도 초기화한다.

## Prepared Target Implementation
- `ActorEntity.PendingEnemyTargetCell`은 준비 중인 공격 또는 이동의 실행 타일을 저장한다.
- `ActorEntity.HasPendingEnemyTargetCell`은 저장된 실행 타일 존재 여부를 나타낸다.
- `AttackTarget`은 공격 준비 시작 시점의 플레이어 위치를 저장하고, 준비 완료 시 저장된 타일만 공격한다.
- 공격 패턴이 방향 기반일 수 있으므로 공격 준비 시작 시점의 방향도 함께 저장하고, 준비 완료 시 저장된 방향으로 공격한다.
- 공격 실행 시점에 플레이어가 저장된 타일에 없으면 공격은 빗나간 것으로 처리하고 준비 상태를 소비한다.
- `WanderMove`는 이동 준비 시작 시점에 이동 가능한 상하좌우 후보 중 하나를 저장하고, 준비 완료 시 저장된 타일로 이동한다.
- `ChaseMove`는 이동 준비 시작 시점에 현재 이동력 기준 최종 도착 타일을 저장하고, 준비 완료 시 저장된 타일로 이동한다.
- 이동력 2 이상인 액터도 최종 타일만 저장한다. 단, 최종 타일은 한 칸씩 이동 가능 여부를 확인한 경로의 결과여야 하며 벽이나 이동 불가 타일을 관통할 수 없다.
- 이동 실행 시점에 저장된 타일이 이동 불가 상태가 되었거나 플레이어/다른 몬스터가 점유 중이면 이동하지 않고 준비 상태를 소비한다.
- 준비 중인 행동은 Behavior Tree에서 우선 처리되어, 준비 완료 전 새 공격/이동 판단으로 덮어쓰지 않는다.

## Prepared Target Visualization
- 준비 중인 공격 타일은 붉은색 계열 마커로 표시한다.
- 준비 중인 이동 타일은 초록색 계열 마커로 표시한다.
- 마커는 액터보다 낮고 바닥보다 높은 정렬 순서로 표시한다.
- 마커 표시는 옵션에서 켜고 끌 수 있다.
- 현재 옵션은 런 중 HUD의 `Options > Prepared target tiles` 토글로 제어한다.

## Deferred Questions
- 이동력이 2 이상일 때 현재 구현은 최종 도착 타일만 저장한다. 이후 경로 중간 타일의 함정, 장판, 충돌 판정이 필요해지면 경로 전체 저장으로 확장할지 결정이 필요하다.
- 저장된 공격 타일에 플레이어가 없을 때 현재 구현은 빗나감으로 처리한다. 이후 빈 타일 공격 연출 또는 범위 공격 잔류 효과가 필요하면 별도 처리 규칙이 필요하다.

## Implementation Notes
- 플레이어 행동 이후 몬스터 턴을 진행하는 기존 흐름은 유지한다.
- 기존 `ActionInterval`은 하위 호환을 위해 남겨두지만, 신규 몬스터 행동 타이밍은 공격/이동 준비 턴을 기준으로 판단한다.
- 공격 범위 판정은 기존 `EnemyAttackPatternDefinition`이 있으면 해당 패턴을 우선 사용하고, 없으면 기본 1칸 직선 공격으로 판정한다.
- 탐지 범위는 공격 패턴, 이동 경로, 맵 구조와 독립적으로 계산한다.
